Veille technologique

Cette veille est centrée sur l’informatique embarquée et le logiciel automobile, dans le cadre de mon BTS SIO option SLAM (mars 2026). Elle analyse les grandes tendances 2025-2026 du secteur automobile numérique, avec un focus sur les compétences SLAM mobilisables.

Le véhicule moderne devient un système logiciel à part entière : architecture zonale, mises à jour OTA, cybersécurité et intelligence artificielle embarquée. Ce rapport met en lumière les enjeux techniques et les applications concrètes pour les développeurs SLAM.

SDV – Software-Defined Vehicle

Le concept de véhicule défini par logiciel transforme l’automobile : architectures centralisées, fonctionnalités activables à la demande, mises à jour OTA et intégration native de l’IA.

  • Architecture zonale
  • Features on Demand
  • Firmware et software update
  • IA embarquée et machine learning

OTA – Over-The-Air

Les mises à jour OTA permettent d’envoyer directement des correctifs et des améliorations vers les véhicules, sans passage en concession.

  • FOTA : firmware, couches basses, OS embarqué
  • SOTA : applications, interface utilisateur, services connectés
  • Architecture : cloud back-end, passerelle télématique, CAN/Ethernet
  • Réglementation : UNECE R155/R156

Cybersécurité automobile

La sécurité des véhicules connectés est devenue un enjeu majeur : les attaques peuvent viser les ECU, les ports OBD-II ou les interfaces sans fil.

  • Risques : OBD-II, Wi-Fi/Bluetooth, infodivertissement, applications mobiles
  • Normes : ISO 21434, UNECE R155, ISO/SAE 21434
  • Approche : Secure by Design, Zero Trust, pentests

IA embarquée

Intelligence artificielle et machine learning sont partout dans l’automobile : conduite autonome, maintenance prédictive, assistant vocal et optimisation de batterie.

  • Conduite autonome (ADAS) : capteurs, traitement temps réel
  • Maintenance prédictive : données télémétriques
  • Assistant vocal : NLP, APIs REST, cloud
  • Cybersécurité IA : détection d’anomalies
